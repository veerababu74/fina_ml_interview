# Gaussian Mixture Models (GMM)

## 1. Core Concept

A **Gaussian Mixture Model** assumes the data is generated by a mixture of K Gaussian distributions. Each cluster is represented by a multivariate Gaussian with its own mean **μ** and covariance **Σ**, weighted by a mixing coefficient **π**.

**Generative model:**
```
p(x) = Σₖ πₖ · N(x | μₖ, Σₖ)

where:
  πₖ  = mixing weight of component k  (Σₖ πₖ = 1, πₖ ≥ 0)
  μₖ  = mean vector of component k
  Σₖ  = covariance matrix of component k
  N(x|μ,Σ) = (2π)^{-d/2}|Σ|^{-1/2} exp(-½(x-μ)ᵀΣ⁻¹(x-μ))
```

GMM performs **soft clustering**: every point has a probability of belonging to each cluster (posterior probability / responsibility).

---

## 2. EM Algorithm — Full Derivation

GMM parameters θ = {πₖ, μₖ, Σₖ}ₖ are learned by maximising the **log-likelihood**:

```
log L(θ) = Σᵢ log [ Σₖ πₖ · N(xᵢ | μₖ, Σₖ) ]
```

Direct maximisation is intractable due to the log-sum. The **Expectation-Maximisation (EM)** algorithm solves this by introducing latent variables zᵢ (cluster assignments).

### E-Step — Compute Responsibilities

For each data point xᵢ and component k, compute the posterior probability (responsibility):

```
r_{ik} = P(z_i=k | x_i, θ) = πₖ · N(xᵢ | μₖ, Σₖ) / Σₗ πₗ · N(xᵢ | μₗ, Σₗ)
```

This is just Bayes' theorem: prior × likelihood / marginal.  
Result: n × K responsibility matrix R where Σₖ r_{ik} = 1 for each i.

### M-Step — Update Parameters

Given the responsibilities from the E-step, maximise the expected complete-data log-likelihood:

**Effective cluster sizes:**
```
Nₖ = Σᵢ r_{ik}
```

**Update mixing weights:**
```
πₖ = Nₖ / n
```

**Update means:**
```
μₖ = (1/Nₖ) Σᵢ r_{ik} · xᵢ
```

**Update covariances:**
```
Σₖ = (1/Nₖ) Σᵢ r_{ik} · (xᵢ − μₖ)(xᵢ − μₖ)ᵀ
```

### EM Convergence

- Repeat E-step and M-step until log-likelihood change < tolerance.
- **Guaranteed to increase log-likelihood** at each iteration (Jensen's inequality).
- Converges to a **local maximum** — sensitive to initialisation.
- Initialise with K-Means++ or run multiple restarts (`n_init`).

## 3. Covariance Types

sklearn's `GaussianMixture` supports four covariance structures trading expressiveness vs. parameters:

| Type | Σₖ Structure | Parameters | Shape |
|------|-------------|-----------|-------|
| **spherical** | σₖ² · I | K scalars | Circular blobs |
| **diag** | diag(σₖ₁², …, σₖd²) | K·d values | Axis-aligned ellipses |
| **tied** | All components share Σ | d² values | Same shape, different positions |
| **full** | Unconstrained Σₖ | K·d² values | Arbitrary ellipses |

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.mixture import GaussianMixture
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler

X, _ = make_blobs(n_samples=300, centers=3, cluster_std=[0.5, 1.2, 0.8], random_state=42)
X = StandardScaler().fit_transform(X)

cov_types = ['spherical', 'diag', 'tied', 'full']
fig, axes = plt.subplots(1, 4, figsize=(18, 4))

for ax, cov_type in zip(axes, cov_types):
    gmm = GaussianMixture(n_components=3, covariance_type=cov_type,
                          n_init=5, random_state=42)
    labels = gmm.fit_predict(X)
    ax.scatter(X[:, 0], X[:, 1], c=labels, cmap='viridis', s=15, alpha=0.7)
    ax.set_title(f'{cov_type}\nBIC={gmm.bic(X):.0f}')
    ax.set_xlabel('Feature 1')

axes[0].set_ylabel('Feature 2')
plt.suptitle('GMM Covariance Types', fontsize=13)
plt.tight_layout()
plt.savefig('gmm_covariance_types.png', dpi=100)
plt.show()
```

**Choosing covariance type:**
- Start with `'full'` — most flexible but most parameters.
- If n is small relative to d, use `'diag'` or `'tied'` to avoid singular matrices.
- Compare with BIC/AIC (see §4).

## 4. Model Selection — BIC and AIC

Both criteria balance log-likelihood fit against model complexity:

```
AIC = 2p − 2·log L(θ̂)
BIC = p·log(n) − 2·log L(θ̂)

where p = number of free parameters, n = number of observations.
```

**BIC penalises complexity more heavily** (by log n vs 2) → prefers simpler models.  
**Rule:** Choose the model with the **lowest** AIC or BIC.

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.mixture import GaussianMixture
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler

X, _ = make_blobs(n_samples=400, centers=4, cluster_std=0.7, random_state=42)
X = StandardScaler().fit_transform(X)

k_range = range(1, 9)
bic_scores  = []
aic_scores  = []

for k in k_range:
    gmm = GaussianMixture(n_components=k, covariance_type='full',
                          n_init=5, random_state=42)
    gmm.fit(X)
    bic_scores.append(gmm.bic(X))
    aic_scores.append(gmm.aic(X))

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(k_range, bic_scores, 'bo-', label='BIC')
ax.plot(k_range, aic_scores, 'rs--', label='AIC')
ax.axvline(x=4, color='green', linestyle=':', label='True k=4')
ax.set_xlabel('Number of Components k')
ax.set_ylabel('Score (lower is better)')
ax.set_title('GMM Model Selection via BIC/AIC')
ax.legend()
plt.tight_layout()
plt.savefig('gmm_bic_aic.png', dpi=100)
plt.show()
print(f"Best k (BIC): {k_range[np.argmin(bic_scores)]}")
print(f"Best k (AIC): {k_range[np.argmin(aic_scores)]}")
```

## 5. Soft vs Hard Assignments

```python
import numpy as np
from sklearn.mixture import GaussianMixture
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler

X, _ = make_blobs(n_samples=200, centers=3, cluster_std=0.8, random_state=42)
X = StandardScaler().fit_transform(X)

gmm = GaussianMixture(n_components=3, covariance_type='full',
                      n_init=5, random_state=42)
gmm.fit(X)

# Soft assignments: probability of belonging to each component
probs = gmm.predict_proba(X)  # shape (n, K)
print("Soft assignments (first 5 points):")
print(np.round(probs[:5], 3))
# e.g. [[0.92, 0.07, 0.01], [0.01, 0.05, 0.94], ...]

# Hard assignments: argmax of soft assignments
labels_hard = gmm.predict(X)  # equivalent to np.argmax(probs, axis=1)
print("\nHard assignments (first 5):", labels_hard[:5])

# Identify uncertain points (max probability < 0.7)
max_prob = probs.max(axis=1)
uncertain_mask = max_prob < 0.70
print(f"\nUncertain points: {uncertain_mask.sum()} ({uncertain_mask.mean()*100:.1f}%)")
```

**When soft assignments matter:**
- Customer lifetime value modelling — weight predictions by membership probability.
- Semi-supervised learning — use confident points as pseudo-labels.
- Anomaly detection — low max-probability indicates an outlier.

## 6. GMM as a Generalisation of K-Means

K-Means is a **special case** of GMM:

| Property | K-Means | GMM |
|----------|---------|-----|
| Assignment | Hard (one cluster) | Soft (probabilities) |
| Cluster shape | Spherical (equal variance) | Elliptical (full covariance) |
| Covariance | σ²·I (implicit, same σ) | Unconstrained per component |
| Objective | Minimise WCSS | Maximise log-likelihood |
| Parameters | μₖ only | μₖ, Σₖ, πₖ |
| EM connection | Hard EM (E-step becomes argmax) | Soft EM |

```python
# Demonstrating K-Means as GMM with spherical, tied covariance
from sklearn.mixture import GaussianMixture
from sklearn.cluster import KMeans
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler
import numpy as np

X, _ = make_blobs(n_samples=300, centers=3, cluster_std=0.5, random_state=42)
X = StandardScaler().fit_transform(X)

# K-Means
km = KMeans(n_clusters=3, init='k-means++', n_init=10, random_state=42)
km_labels = km.fit_predict(X)

# GMM with spherical tied covariance ≈ K-Means
gmm_km = GaussianMixture(n_components=3, covariance_type='spherical',
                         n_init=10, random_state=42)
gmm_km_labels = gmm_km.fit_predict(X)

# Full GMM
gmm_full = GaussianMixture(n_components=3, covariance_type='full',
                            n_init=10, random_state=42)
gmm_full_labels = gmm_full.fit_predict(X)

print("K-Means inertia:", round(km.inertia_, 2))
print("GMM spherical log-likelihood:", round(gmm_km.score(X) * len(X), 2))
print("GMM full    log-likelihood:", round(gmm_full.score(X) * len(X), 2))
```

## 7. Anomaly Detection with GMM

```python
import numpy as np
from sklearn.mixture import GaussianMixture
from sklearn.preprocessing import StandardScaler

np.random.seed(42)
X_normal  = np.random.multivariate_normal([0, 0], [[1, 0.5],[0.5, 1]], 500)
X_anomaly = np.random.uniform(-5, 5, (20, 2))
X = np.vstack([X_normal, X_anomaly])
X = StandardScaler().fit_transform(X)

gmm = GaussianMixture(n_components=2, covariance_type='full',
                      n_init=5, random_state=42)
gmm.fit(X[:500])  # train on normal data only

log_probs = gmm.score_samples(X)  # log p(xᵢ) under the fitted GMM
threshold = np.percentile(log_probs[:500], 1)  # 1st percentile of training data

anomalies = X[log_probs < threshold]
print(f"Anomalies detected: {len(anomalies)}")
print(f"True anomaly recall: {((log_probs[500:] < threshold).sum()) / 20:.2%}")
```

## 8. Interview Q&A

**Q1: Explain the E-step and M-step of EM for GMMs.**  
A: **E-step:** Compute the responsibility r_{ik} = posterior probability that point i belongs to component k, using Bayes' theorem with current parameters. **M-step:** Update parameters (πₖ, μₖ, Σₖ) by maximising the expected complete-data log-likelihood, treating responsibilities as soft counts. Repeat until convergence. Each full iteration is guaranteed to non-decrease the log-likelihood (by Jensen's inequality).

**Q2: How does GMM differ from K-Means?**  
A: GMM is a probabilistic model with soft cluster assignments (each point has a probability of belonging to each cluster). K-Means uses hard assignments. GMM can model elliptical clusters via full covariance matrices; K-Means assumes spherical clusters. K-Means is essentially GMM with spherical tied covariance and hard E-step (responsibility is 0 or 1).

**Q3: How do you choose the number of components K?**  
A: Use **BIC** (Bayesian Information Criterion) — fit GMMs for various K and pick the K that minimises BIC. BIC penalises model complexity more than AIC. For clustering, BIC is generally preferred. Also use AIC as a sanity check; if BIC and AIC disagree, favour BIC.

**Q4: What is the covariance_type parameter and when should each be used?**  
A: `spherical`: circular clusters, fewest params. `diag`: axis-aligned ellipses, good when features are independent. `tied`: all clusters same shape, different positions. `full`: arbitrary ellipses, most flexible but requires most data. Use `full` by default; fall back to simpler types when n is small or the model is singular.

**Q5: Can GMM overfit? How do you prevent it?**  
A: Yes — with `full` covariance and many components, a component can collapse onto a single point (covariance → 0, likelihood → ∞). Fix: (1) use `reg_covar` parameter in sklearn to add regularisation; (2) use BIC/AIC to limit K; (3) use simpler covariance types. sklearn adds `reg_covar=1e-6` by default.

**Q6: How is EM guaranteed to converge?**  
A: EM maximises a lower bound (ELBO) on log-likelihood at each step. The E-step tightens the bound; the M-step maximises it. This guarantees monotonic non-decrease of log-likelihood. However, it may converge to a local maximum — run multiple restarts with `n_init`.

## 9. Common Pitfalls

1. **Singular covariance matrix** — too few points per component or too many components; increase `reg_covar` or reduce K.
2. **Local maxima** — EM is sensitive to initialisation; always set `n_init ≥ 5`.
3. **Choosing K by log-likelihood alone** — it always increases with K; use BIC/AIC.
4. **Not scaling features** — GMM fits are affected by feature scale.
5. **Treating GMM components as clusters** — not all components correspond to meaningful clusters (some may model noise or tails).
6. **Assuming GMM works on any distribution** — data with heavy tails or multi-modal components within a single cluster will not fit well.
7. **Forgetting `predict_proba` gives soft assignments** — use these for uncertainty-aware downstream tasks.

## 10. Quick Reference Cheat Sheet

```
Model:    p(x) = Σₖ πₖ · N(x | μₖ, Σₖ)
Training: EM algorithm (E-step: compute responsibilities, M-step: update params)
Select K: BIC (lower = better); AIC as secondary
Cov types: spherical < diag < tied < full  (complexity order)
Init:     K-Means++ style; use n_init ≥ 5
Soft:     predict_proba(X) → n×K responsibility matrix
Hard:     predict(X) → argmax(predict_proba(X))
Anomaly:  score_samples(X) → log p(xᵢ); low score = anomaly
vs K-Means: GMM is probabilistic; handles elliptical clusters; soft assignments

sklearn quick start:
  from sklearn.mixture import GaussianMixture
  gmm = GaussianMixture(n_components=4, covariance_type='full',
                        n_init=5, random_state=42)
  gmm.fit(X_scaled)
  labels      = gmm.predict(X_scaled)        # hard
  probs       = gmm.predict_proba(X_scaled)  # soft
  bic_score   = gmm.bic(X_scaled)
```
